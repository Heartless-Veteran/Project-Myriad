## Architecture Overview

**Project Myriad** follows **Clean Architecture** principles with clear separation of concerns:

- **Core Modules**: Domain logic, data access, and shared UI components
- **Feature Modules**: Self-contained feature implementations
- **Application Layer**: Main app coordination and security

**Key Principles:**
- **Dependency Inversion**: Core modules don't depend on implementation details
- **Single Responsibility**: Each module has a clear, focused purpose  
- **Testability**: Clean interfaces enable comprehensive testing
- **Maintainability**: Modular structure supports independent development
